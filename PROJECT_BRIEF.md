# デジタル社内掲示板 - プロジェクト概要

## 目的
中企業の取引先（工場）にある壁の掲示板（社内報・安全月間・イベント案内等）をデジタル化する。
会社への企画提案用のデモを作成する。

## 背景・課題
- 紙の掲示板は見に行かないと読めない
- 張り替えが手間
- 見たかどうか確認できない
- → PWAなら、スマホで見られる・更新が簡単・QR貼るだけで導入できる

## 画面構成（2パターン）

### 1. スマホ用 (index.html)
- 対象: 社員個人のスマホ
- レイアウト: 縦長1カラム、記事一覧 → タップで詳細表示
- イメージ: ニュースアプリ的

### 2. サイネージ用 (signage.html)
- 対象: 壁掛けタブレット・モニター（紙の掲示板の代替）
- レイアウト: 横長マルチカラム、複数記事を同時表示
- 動作: 30秒等で自動切替、タップで全文表示
- イメージ: 駅のインフォメーション的

### 共通事項
- データソースは1つ（同じ記事データ）
- URLが違うだけで表示を切り分ける
- 企画提案時の推しポイント:「1回の投稿でスマホにもモニターにも同時反映。二重管理不要」

## 開発フェーズ

### フェーズ1: デモ用（★今回作るのはここ）
1. HTMLで掲示板画面を作る（スマホ用 + サイネージ用）
2. 記事データはHTML内にベタ書き（JSON的に埋め込み）
3. PWA化（manifest.json, sw.js, アイコン）
4. GitHub Pagesで公開
5. スマホ・タブレットで「こんな感じです」と見せる

### フェーズ2: 実運用（企画が通ったら）
1. スプレッドシートにデータを移す
2. PWAからスプレッドシートを読んで表示（「ウェブに公開」機能 or GAS中継）
3. Googleフォームで投稿 → スプレッドシートに自動蓄積 → PWA表示
4. プッシュ通知が必要ならFirebase等を検討

## 技術構成

```
D:\DigitalKeijiban\
  ├── index.html              ← スマホ用PWA
  ├── signage.html            ← サイネージ用PWA
  ├── manifest.json           ← PWA定義（スマホ用）
  ├── manifest-signage.json   ← PWA定義（サイネージ用、必要に応じて）
  ├── sw.js                   ← Service Worker（Network First）
  └── icons/
      ├── icon-192.png
      └── icon-512.png
```

- GitHub Pages で公開
- Service Worker は Network First 戦略（オンライン時は常に最新取得）
- manifest.json を分ければ別アプリとしてインストール可能
- サイネージ用はデモ段階ではブラウザ全画面表示でOK

## デモ用の記事サンプル（例）
- 安全月間のお知らせ（カテゴリ: 安全）
- 社内BBQ開催のお知らせ（カテゴリ: イベント）
- 健康診断の受付開始（カテゴリ: 総務）
- 新入社員の紹介（カテゴリ: 社内報）
- 駐車場利用ルールの変更（カテゴリ: お知らせ）

## 前プロジェクトからの学び（あみだくじPWA化）
- PWA化の手順は経験済み（manifest.json, sw.js, メタタグ）
- Network First にすればキャッシュ問題が減る
- HTMLを大幅修正するとPWAタグが消える事故に注意
- インストール案内は beforeinstallprompt + フォールバックテキスト
- ゲーム開始等の画面遷移時にバナー非表示のフラグ制御が必要
